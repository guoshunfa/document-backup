                    # MongoDB shell语法

                    # 命令行选项

                    该mongo (opens new window)外壳可以与众多的选择开始。有关所有可用选项的详细信息，请参见 mongo shell (opens new window)页面。

                    下表显示了一些常见的选项mongo (opens new window)：

                    选项 描述
                    --help (opens new window) 显示命令行选项
                    --nodb (opens new window) 在mongo (opens new window)不连接数据库的情况下启动Shell。要稍后连接，请参阅“ 打开新连接” (opens new window)。
                    --shell (opens new window) 与JavaScript文件（即 (opens new window)）结合使用，可 mongo (opens new window)在运行JavaScript文件后继续在 Shell中运行。有关示例，请参见JavaScript文件 (opens new window)。

                    # 命令助手

                    该mongo (opens new window)外壳提供了各种帮助。下表显示了一些常见的帮助方法和命令：

                    帮助方法和命令 描述
                    help 显示帮助。
                    db.help() (opens new window) 显示有关数据库方法的帮助。
                    db.<collection>.help() 在收集方法上显示帮助。该<collection>可以是现有的集合或不存在的集合的名称。
                    show dbs 打印服务器上所有数据库的列表。该操作与listDatabases (opens new window)命令相对应。如果部署使用访问控制运行，则该操作将根据用户权限返回不同的值。有关详细信息，请参见 listDatabases行为 (opens new window)。
                    use <db> 将当前数据库切换到<db>。该mongo (opens new window)shell变量db被设置为当前数据库。
                    show collections 打印当前数据库的所有集合的列表。也可以看看显示收藏 (opens new window)
                    show users 打印当前数据库的用户列表。
                    show roles 打印当前数据库的所有角色的列表，包括用户定义角色和内置角色。
                    show profile 打印耗时1毫秒或更长时间的五个最新操作。有关更多信息，请参见数据库分析器 (opens new window)上的文档。
                    show databases 打印所有可用数据库的列表。该操作与listDatabases (opens new window)命令相对应。如果部署使用访问控制运行，则该操作将根据用户权限返回不同的值。有关详细信息，请参见 listDatabases行为 (opens new window)。
                    load() 执行一个JavaScript文件。有关 更多信息，请参见 mongo Shell的编写脚本 (opens new window)。

                    # Shell JavaScript基本操作

                    该mongo (opens new window)外壳程序提供了用于数据库操作的 JavaScript API (opens new window)。

                    在mongo (opens new window)外壳程序中，db是引用当前数据库的变量。该变量自动设置为默认数据库，test或者在使用切换当前数据库时设置。use <db>

                    下表显示了一些常见的JavaScript操作：

                    JavaScript数据库操作 描述
                    db.auth() (opens new window) 如果以安全模式运行，请对用户进行身份验证。
                    coll = db.<collection> 将当前数据库中的特定集合设置为一个变量 coll，如以下示例所示：coll = db.myCollection;您可以myCollection使用变量对变量执行操作，如以下示例所示：coll.find();
                    db.collection.find() (opens new window) 查找集合中的所有文档并返回一个游标。有关更多信息和示例，请参见db.collection.find() (opens new window)和 查询文档 (opens new window)。有关在外壳 (opens new window)中处理游标的信息，请参阅在mongo Shell中迭代游标 (opens new window)mongo (opens new window)。
                    db.collection.insertOne() (opens new window) 将新文档插入集合中。
                    db.collection.insertMany() (opens new window) 将多个新文档插入集合中。
                    db.collection.updateOne() (opens new window) 更新集合中的单个现有文档。
                    db.collection.updateMany() (opens new window) 更新集合中的多个现有文档。
                    db.collection.save() (opens new window) 插入新文档或更新集合中的现有文档。
                    db.collection.deleteOne() (opens new window) 从集合中删除单个文档。
                    db.collection.deleteMany() (opens new window) 从集合中删除文档。
                    db.collection.drop() (opens new window) 完全删除或删除集合。
                    db.collection.createIndex() (opens new window) 如果索引不存在，则在集合上创建一个新索引；否则，该操作无效。
                    db.getSiblingDB() (opens new window) 使用相同的连接返回对另一个数据库的引用，而无需显式切换当前数据库。这允许跨数据库查询。

                    有关在shell中执行操作的更多信息，请参见：

                    • MongoDB CRUD操作 (opens new window)
                    • mongo Shell方法 (opens new window)

                    # 查询

                    在mongo (opens new window)Shell中，使用find() (opens new window)和findOne() (opens new window) 方法执行读取操作 。

                    该find() (opens new window)方法返回一个游标对象，mongo (opens new window)shell对其进行迭代以在屏幕上打印文档。默认情况下，mongo (opens new window)打印前20个结果 mongo (opens new window)。shell将提示用户“ ”以继续迭代下20个结果。Type it

                    下表提供了mongo (opens new window)Shell 中的一些常见读取操作 ：

                    读取操作 描述
                    db.collection.find() (opens new window) <query>在集合中找到符合条件的文档。如果<query>未指定标准或该标准为空（即{}），则读取操作将选择集合中的所有文档。下面的示例选择users 集合中name字段等于的文档"Joe"：coll = db.users; coll.find( { name: "Joe" } );有关指定<query>条件的更多信息，请参阅“ 指定相等条件” (opens new window)。
                    db.collection.find(, ) (opens new window) 查找符合<query>条件的文档，然后仅返回中的特定字段<projection>。以下示例从集合中选择所有文档，但仅返回该name字段和该_id字段。将 _id始终返回，除非明确指定有来无回。coll = db.users; coll.find( { }, { name: true } );有关指定的更多信息<projection>，请参见 要从查询返回的项目字段 (opens new window)。
                    db.collection.find().sort() (opens new window) 返回指定结果。<sort order>以下示例从集合中选择所有文档，并返回按name字段升序（1）排序的结果。使用-1降序排序：coll = db.users; coll.find().sort( { name: 1 } );
                    db.collection.find().sort() (opens new window) 返回<query>与指定条件中的条件相符的文档。<sort order>
                    db.collection.find( ... ).limit( ) (opens new window) 将结果限制为<n>行。如果只需要一定数量的行以获得最佳性能，则强烈建议使用。
                    db.collection.find( ... ).skip( ) (opens new window) 跳过<n>结果。
                    db.collection.count() (opens new window) 返回集合中的文档总数。
                    db.collection.find().count() (opens new window) 返回与查询匹配的文档总数。在count() (opens new window)忽略limit() (opens new window)和skip() (opens new window)。例如，如果有100条记录匹配，但限制为10， count() (opens new window)则将返回100。这比迭代自己的速度更快，但仍然需要时间。
                    db.collection.findOne() (opens new window) 查找并返回一个文档。如果找不到，则返回null。以下示例在users 集合中选择一个与name字段匹配的文档"Joe"：coll = db.users; coll.findOne( { name: "Joe" } );在内部，该findOne() (opens new window) 方法是find() (opens new window)带有的方法limit(1) (opens new window)。

                    有关更多信息和示例，请参阅查询文档 (opens new window)文档。请参阅查询和投影运算符 (opens new window)以指定其他查询运算符。
